HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • 'pwd' displays the current working directory
  • 'ls' displays the list of files and folders in the current directory
  • 'cd' navigates between directories in the path
  • 'cd ..' navigates backwards, one level at a time, through the path of your directories
  • 'mkdir' creates a new directory
  • 'touch' creates a file inside a directory

CSS

  • Selector: defines the element or attributes to which the rules, or declarations, will apply.
  • Declaration: the CSS property we want to apply and the value of the property.
  • Margin: indicates how much space we want around the outside of an element. Not visible on the page.
  • Padding: Adds space around the content inside an element. Increasing padding makes the element bigger.
  • Adding a '*' in the declaration box declares that ALL of the elements on the page will have the style properties margin and padding set as the number you put.

Git

  • git status: checks what branch we are currently o
  • git checkout -b branch-name: creates a new branch and switches to it.

Github Workflow

  • Make sure you are on the branch you are pushing. To check: git status
  • Stage your changes by using: git add -A ; this commits all files in the working branch
  • To commit changes to repository use: git commit -m "add your note about changes here"
  • Make sure current branch is up to date before opening a pull request: git pull origin main
  • Push up all changes by using: git push origin feature/[branch title]
  • Merge feature branch into base branch: go to repository, select the feature branch you are working on
  • Select Pull Request - New Pull Request
  • Merge Pull Request: feature branch should now show up on main branch
  • Update local repository with new merged-in feature: git checkout main, git pull origin main
  • Close HTML issue on Github; Issues - Select the one you want to close and select 'Close Issue'

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • var (variable) is how we give a variable a name, must be unique.
  • = assignment operator to assign specific data that we want stored in our variable.
